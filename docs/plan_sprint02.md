# Sprint 02 – Planning Document

## 1. Sprint Goal
El objetivo de este sprint es implementar la lógica principal de la aplicación Travel Planner gestionando operaciones CRUD para los viajes y sus actividades bajo el patrón MVVM. Para ello, se creará una base de datos falsa (Fake Database) en memoria que permita comprobar el correcto funcionamiento y flujo de la app.
